a simple shared memory system for llms

rodigu

2026-06-19 21:30

i wanted an easy way to share context between sessions for the members of my team.

the basic first requirements i had for this system were:

  • crud: the ability to perform all 4 basic operations on the data
  • easy search: a simple, straight-forward way to search through the data so llms can easly retrieve relevant context

i did consider using a vector database, implementing a rag. but rags are not free to develop, run and maintain (talking mainly about infra costs here). i also think they would have been overkill1, a simple grep would sufice on well tagged data.

in my experience, llms work best with short, well structured information. if they are given too much room to halucinate, they will do it2.
